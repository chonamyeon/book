# Archiview 원스톱 팟캐스트 자동화 시스템

---

## 서비스 정의 & 콘텐츠 철학

### 우리는 무엇을 만드는가

**이카이뷰(Archiview)는 책 소개 사이트가 아닙니다.**

우리가 만드는 것은 **오리지널 팟캐스트 콘텐츠**입니다.
책을 소재로 삼되, 책을 단순히 요약하거나 소개하는 것이 아니라
책 속의 아이디어를 가지고 **토론하고, 논쟁하고, 삶에 연결**짓는 것이 핵심입니다.

### 핵심 콘셉트

> **"아침 출근길, 웃으면서 듣는 책 토론 팟캐스트"**

- 출퇴근 시간, 점심시간 — 직장인의 틈새 시간을 공략
- 진지하고 딱딱한 책 이야기가 아니라 **밝고 유쾌한 분위기**로 진행
- 제임스와 스텔라가 **친구처럼 웃으며 대화**하는 토론 형식
- 책 내용에서 출발해 **직장생활 · 사회생활 · 인생의 인사이트**로 확장
- 듣는 사람이 "아, 이게 내 얘기네"라고 공감하게 만드는 콘텐츠

### 차별화 전략

| 일반 책 소개 서비스 | 이카이뷰 |
|-------------------|---------|
| 책 내용 요약 | 책을 소재로 한 **오리지널 토론** |
| 정보 전달 중심 | **인사이트 + 공감 + 재미** |
| 독서가 대상 | 책 안 읽는 직장인도 즐길 수 있음 |
| 수동적 청취 | 대화 속으로 끌려들어가는 몰입감 |
| 단순 오디오북 | 삶과 연결되는 **오리지널 콘텐츠** |

### 콘텐츠가 다루는 영역

책 내용을 바탕으로 아래 주제들로 확장합니다:
- **직장생활** — 상사 관계, 번아웃, 커리어 결정, 회의 문화
- **사회생활** — 인간관계, 설득, 갈등 해결, 커뮤니케이션
- **인생사** — 선택의 기로, 후회, 성장, 의미 찾기
- **성공 인사이트** — 성공한 사람들의 사고방식과 행동 패턴

### 팟캐스트 톤 앤 매너

- 두 진행자가 **1:1로 번갈아 토론** (제임스 ↔ 스텔라)
- [웃음] 같은 지시문 없이 **텍스트 자체에서 자연스러운 유머** 구현
- 상투적 인사말 없이 **본론부터 바로 시작** (직장인 시간 존중)
- 어렵고 무거운 개념도 **일상적 비유와 직장인 경험**으로 풀어냄
- 45턴 이상의 충분한 대화로 **출퇴근 시간(20~30분) 완주** 가능

## 원스톱 자동화 파이프라인

> 도서 파일 1개 투입 → 이북 + 팟캐스트 완성까지 완전 자동 생성

---

## 전체 파이프라인 흐름

```
① 도서 파일 투입 (ebook_inputs/)
        ↓
② 텍스트 자동 추출 (PDF / TXT)
        ↓
③ Google Books API → 표지 자동 검색 & 저장
        ↓
④ Gemini 2.5 Flash → 팟캐스트 대본 생성 (45턴↑)
        ↓
⑤ Gemini 2.5 Flash → 이북 리뷰 생성 (4,000자↑)
        ↓
⑥ TTS 생성 (두 가지 방식)
   ├── 일반도서  (45턴↑) → gemini-2.5-flash-preview-tts
   └── 오리지널도서 (40턴↑) → gemini-2.5-pro-preview-tts
        ↓
⑦ ffmpeg 오디오 병합 → MP3 변환
        ↓
⑧ celebrities.js 자동 연동 (podcastFile, cover, review)
        ↓
⑨ bookScripts.js 자동 업데이트
        ↓
⑩ npm run deploy → 실서버 자동 배포 (Firebase Hosting)
```

## 실행 방법

### 1. 도서 파일 준비
```
the-archive/
└── ebook_inputs/
    ├── sapiens.txt          ← 도서 ID = sapiens
    ├── atomic-habits.pdf    ← 도서 ID = atomic-habits
    └── norwegian-wood.txt   ← 도서 ID = norwegian-wood
```
- 파일 이름 = 도서 ID (영문 소문자, 하이픈 사용)
- `.txt` 또는 `.pdf` 형식 지원

### 2. 파이프라인 실행
```bash
# 특정 책 1권만 처리 (권장)
node scripts/auto_podcast_pipeline.mjs norwegian-wood

# 전체 처리 (ebook_inputs의 모든 책)
node scripts/auto_podcast_pipeline.mjs
```

### 2-1. 이어받기 (TTS 중단 후 재시작)
```bash
# 동일 명령어 재실행 → 자동 이어받기
node scripts/auto_podcast_pipeline.mjs norwegian-wood
# → 대본(script.json) 기존 파일 재사용
# → 완료된 turn_N.wav 자동 건너뜀
# → 미완성 턴부터 재개 (이어받기 모드 출력됨)
```

### 3. 출력 결과
```
public/
├── audio/
│   └── {bookId}.mp3          ← 완성된 팟캐스트
└── images/covers/
    └── {bookId}.jpg          ← 도서 표지

final_podcast/
├── {bookId}_script.json      ← 생성된 대본 (JSON)
├── {bookId}_script.txt       ← 생성된 대본 (텍스트)
└── {bookId}_review.txt       ← 생성된 이북 리뷰

src/data/
├── celebrities.js            ← podcastFile, cover, review 자동 업데이트
└── bookScripts.js            ← 팟캐스트 대본 자동 추가
```

## 에러 처리 & 이어받기

### TTS 중간 실패 시 (이어받기 지원)
- `temp_audio/{bookId}/turn_N.wav` 파일이 이미 존재하면 자동으로 건너뜀
- 실패한 turn부터 다시 시작 → 처음부터 다시 생성 불필요

### 병합 실패 시
- 생성된 turn 파일들로 `{bookId}_partial.mp3` 생성 (부분 결과물)
- 로그: `factory_log.txt`

### 대본 생성 실패 시
- 기존 `final_podcast/{bookId}_script.json`이 있으면 해당 대본으로 TTS 계속 진행

## 이북 시스템 기술 정보

- **컴포넌트**: `src/pages/ReviewDetail.jsx`
- **Flipbook 라이브러리**: `react-pageflip` — 실제 책장을 넘기는 경험 구현
- **페이지 추적**: `onFlip` 이벤트로 현재 페이지 인덱스 실시간 추적
- **텍스트 분할(Chunking)**: 긴 리뷰 텍스트는 페이지 넘침 방지를 위해 **약 400자 단위**로 자동 분할

> **리뷰 업데이트 시 주의**: 도서 리뷰는 `src/data/celebrities.js`에서 수정합니다.
> 4,000자 이상의 장문 리뷰도 Chunking으로 자동 처리되므로 분량 제한 없이 작성 가능합니다.

## 고퀄리티 롱폼 독후감 작성 전략 (4,000자 이상)

### 1단계: 준비 — 읽으면서 메모하기

분량이 많은 글은 기억력에만 의존할 수 없습니다. 책을 읽으며 다음을 기록하세요.

- **인상 깊은 문장 + 페이지 번호**: 인용구로 활용하여 신뢰도와 분량 확보
- **순간적인 감정·의문**: "왜 여기서 이렇게 행동했지?" 같은 질문이 훌륭한 분석 소재
- **사회적 배경·저자 이력**: 책 외부 정보를 결합하면 글의 깊이가 달라짐

---

### 2단계: 논리적 구조 설계 — 분량 배분표

| 순서 | 항목 | 내용 요약 | 권장 분량 |
| :--- | :--- | :--- | :--- |
| 1 | 서론 | 책을 선택한 동기, 첫인상, 사회적 배경 | 약 500자 |
| 2 | 줄거리 요약 | 단순 나열이 아닌 핵심 갈등 위주 | 약 800자 |
| 3 | 심층 분석 1 | 인물 분석 및 상징적 의미 해석 | 약 800자 |
| 4 | 심층 분석 2 | 저자의 철학 및 사회적 메시지 연결 | 약 800자 |
| 5 | 개인적 성찰 | 내 삶과의 연결, 변화된 생각 | 약 800자 |
| 6 | 결론 | 총평, 추천 대상, 마무리 문구 | 약 300자 |

---

### 3단계: 세부 작성 전략 — 분량 늘리는 법

1. **서론**: '나'와 '책'의 만남을 구체화
   - 당시 처했던 상황·고민을 언급 (예: "취업 준비로 자존감이 낮아진 시기에...")

2. **줄거리**: 장면을 묘사하듯 쓰기
   - 중요 사건의 전후 맥락과 긴장감, 주인공의 심리 변화를 서술

3. **인용구 활용 공식** (가장 중요)
   ```
   [인용문] + [문장의 의미 해석] + [내 생각] + [현실 세계로의 확장]
   ```
   → 최소 5~10문장 이상 덧붙이기

4. **외부 자료 연결**: 비슷한 주제의 영화·뉴스·다른 책과 비교 분석

5. **'만약에' 기법**: "만약 주인공이 다른 선택을 했다면?" 가정으로 한 섹션 구성

---

### 4단계: 실전 팁

- **소제목 붙이기**: 4~5개 소제목으로 가독성 확보
- **문체**: 풍부한 수식어와 연결어(게다가, 요컨대 등)를 사용하는 문어체
- **문답 형식**: 독자·저자에게 질문을 던지고 스스로 답하기
- **구체적 에피소드**: 시간, 장소, 상황을 상세하게 묘사

### 5단계: 글쓰기 개요 예시 (『오만과 편견』)

- **서론**: 19세기 영국 배경 + 제인 오스틴 언급
- **본론 1**: 엘리자베스 ↔ 다아시 만남과 갈등
- **본론 2**: '오만' 키워드로 본 다아시 성격 분석 + 나의 편견 반성
- **본론 3**: 당시 여성 결혼관 vs. 현대 결혼관 비교
- **본론 4**: 청혼 장면 인용 + 세밀한 심리 묘사
- **결론**: 진정한 소통에 대한 나의 정의와 최종 소감

## 이북 리뷰 작성 원칙 (10가지)

1. **반복 금지**: 모든 문장은 고유한 의미를 가짐. 채우기용 문구(filler) 절대 금지
2. **비율**: 도서 내용 요약 30% + 나의 독창적 성찰 70%
3. **자연스러운 분량**: 억지로 채우지 않고, 내용의 깊이에 따라 4,000자 수준의 밀도 높은 텍스트 구성
4. **독자 시점**: 실제 이북을 읽는 듯한 몰입감을 주는 문체
5. **가독성**: 이북 뷰어에서 읽힐 것을 고려하여 띄어쓰기·단락 구분을 깔끔하게
6. **숨기기**: "도서요약 (30%)", "성찰 (70%)" 같은 내부 구조 설명은 본문에 노출 금지
7. **출처 표기**: 글 맨 앞 또는 뒤에 아래 형식으로 명시
   ```
   참고 도서: 도서명 / 저자: 저자명 / 출판사: 출판사명
   ```
8. **직접 인용 표기**: 책의 특정 문장을 그대로 옮길 때는 큰따옴표(" ") 사용 필수
9. **내 생각 중심**: 단순 책 소개가 아닌 **책을 읽는 나의 생각**을 중심으로 서술
   - 책 내용과 내 생각이 어떻게 연결되는지를 풀어내기
   - 단순 소개는 저작권 문제 소지 있음 → 내 성찰·관점 위주로 작성
10. **이북 오프닝**: 추천인이 감명 깊게 읽었다고 언급한 특정 구절을 중심으로 시작
    - 인상적인 인용문으로 시작하면 독자의 몰입도 향상
